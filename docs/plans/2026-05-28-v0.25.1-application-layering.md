# textus 0.25.1 — Application Layering Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Finish the layering work that ADRs 0013 and 0014 started. Stop passing `store:` into the Application layer, split the read/write halves of `EnvelopeIO`, and carve `Manifest` into the four roles it actually plays (Data, Resolver, Policy, Rules). The result: every use-case constructor declares exactly which slice of the store it needs, and no application class can reach the composition root.

**Architecture:** Three independent refactors composed into one release because they touch overlapping constructors. Land in order — `Manifest` carving first (it changes the type that `Ports` will hold), then `Ports`, then `EnvelopeIO` split — so each task lands on the final shape of the layer it sits on. No new infra. No new public verbs. No wire-format change.

**Tech Stack:** Ruby 3.3+, RSpec, existing `Textus::Manifest`, existing `Application::Writes::*`, existing `Application::Reads::*`, existing `Operations`. Zeitwerk autoloads the new files without inflector changes.

**ADRs:** [0016 — Application Ports](../architecture/decisions/0016-application-ports-value.md), [0017 — EnvelopeIO split](../architecture/decisions/0017-envelope-io-split.md), [0018 — Manifest carving](../architecture/decisions/0018-manifest-carving.md).

**Scope:** Internal refactor only. Protocol stays `textus/3`. Wire envelopes unchanged. Patch bump because no public CLI / Operations surface changes; the only public-ish renames are the hook DSL kwarg (`store:` → `ports:`) and the Manifest deprecation shims, both with one-cycle bridges that keep existing user code working.

**Out of scope:** ADR 0019 (Hooks::Bus split into RpcRegistry + EventBus), ADR 0020 (PublishStrategy extraction from Manifest entries), ADR 0021 (CLI registry demagic + Store#boot!). These ship in 0.26.0.

## File Structure

**New files:**
- `lib/textus/manifest/data.rb` — raw declaration value object (was `Manifest` core state).
- `lib/textus/manifest/policy.rb` — role/zone authority view.
- `lib/textus/application/ports.rb` — `Data.define` value bundling six collaborators.
- `lib/textus/application/writes/envelope_reader.rb` — read half.
- `lib/textus/application/writes/envelope_writer.rb` — write half.
- `spec/manifest/data_spec.rb`
- `spec/manifest/policy_spec.rb`
- `spec/application/ports_spec.rb`
- `spec/application/writes/envelope_reader_spec.rb`
- `spec/application/writes/envelope_writer_spec.rb`

**Modified files:**
- `lib/textus/manifest.rb` — becomes a `Data.define(:data, :resolver, :policy, :rules)` composition record with deprecation shims for legacy top-level methods.
- `lib/textus/operations.rb` — builds `Ports` once; every use-case factory passes `ports:` instead of individual ports.
- `lib/textus/store.rb` — no signature change; `Operations.for(self)` continues to work.
- `lib/textus/application/writes/put.rb` — takes `ports:` + `writer:`; drops `manifest:`, `envelope_io:`, `bus:`.
- `lib/textus/application/writes/delete.rb` — same shape as Put.
- `lib/textus/application/writes/mv.rb` — takes `ports:`, `reader:`, `writer:`.
- `lib/textus/application/writes/accept.rb` — takes `ports:`, `writer:`.
- `lib/textus/application/writes/reject.rb` — takes `ports:`, `writer:`.
- `lib/textus/application/writes/publish.rb` — takes `ports:`; drops `store:`.
- `lib/textus/application/refresh/worker.rb` — takes `ports:`; `store:` removed.
- `lib/textus/application/refresh/all.rb` — takes `ports:`.
- `lib/textus/application/refresh/orchestrator.rb` — takes `ports:`.
- `lib/textus/application/reads/*.rb` — every reader takes `ports:` and pulls the slice it needs (`ports.manifest.resolver`, `ports.manifest.rules`, etc).
- `lib/textus/application/restructure/*.rb` — takes `ports:`; the `store:` kwarg goes away.
- `lib/textus/application/projection.rb` — hook RPC bridge: accepts both `store:` and `ports:` kwargs from user callables until 0.26.0.
- `lib/textus/hooks/bus.rb` — `filter_kwargs` learns to inject `ports:` if the callable declares it.
- `lib/textus/application/writes/envelope_io.rb` — **deleted** at the end of Task 3.
- `spec/**/*.rb` — any spec that constructs application classes directly is updated to pass `ports:` and `writer:`.
- `lib/textus/version.rb` — bump to `0.25.1`.
- `CHANGELOG.md`, `Gemfile.lock`.

- [ ] **Step 4: Reshape `Manifest` into a composition record**

```ruby
Manifest = Data.define(:data, :resolver, :policy, :rules) do
  def self.load(root)
    manifest_path = File.join(root, "manifest.yaml")
    raise IoError.new("manifest not found: #{manifest_path}") unless File.exist?(manifest_path)
    raw = YAML.safe_load_file(manifest_path, aliases: false)
    check_version!(raw, manifest_path)
    data = Manifest::Data.parse(raw, root: root)
    new(
      data:     data,
      resolver: Manifest::Resolver.new(data),
      policy:   Manifest::Policy.new(data),
      rules:    Manifest::Rules.parse(raw["rules"] || []),
    )
  end

  # … parse(yaml_text, root:) similar … check_version! preserved
end
```

- [ ] **Step 2: Create the value object**

```ruby
module Textus
  module Application
    Ports = Data.define(
      :manifest, :file_store, :schemas, :audit_log,
      :event_bus, :rpc_registry, :root,
    ) do
      def self.from_store(store)
        new(
          manifest:     store.manifest,
          file_store:   store.file_store,
          schemas:      store.schemas,
          audit_log:    store.audit_log,
          event_bus:    store.bus,
          rpc_registry: store.bus,
          root:         store.root,
        )
      end
    end
  end
end
```

- [ ] **Step 3: Wire `Operations` to use Ports**

```ruby
def self.for(store, role: Role::DEFAULT, correlation_id: nil, dry_run: false)
  new(
    ctx:   Application::Context.build(role: role, correlation_id: correlation_id, dry_run: dry_run),
    ports: Application::Ports.from_store(store),
  )
end

def initialize(ctx:, ports:)
  @ctx   = ctx
  @ports = ports
  @authorizer = Domain::Authorizer.new(manifest: @ports.manifest)
end
```

Delete the seven individual instance variables (`@manifest`, `@file_store`, `@schemas`, `@audit_log`, `@bus`, `@root`, `@store`). Update every factory method (`put`, `delete`, `mv`, `accept`, `reject`, `publish`, `get`, `get_or_refresh`, `list`, `where`, `uid`, `schema_envelope`, `deps`, `rdeps`, `published`, `stale`, `audit`, `blame`, `policy_explain`, `freshness`, `pulse`, `validate_all`, `refresh`, `refresh_all`, `key_mv_prefix`, `key_delete_prefix`, `zone_mv`, `rule_lint`, `migrate`) to pass `ports: @ports`.

- [ ] **Step 4: Update every application class**

For each file under `application/{writes,reads,refresh,restructure}/`:

1. Replace constructor kwargs `manifest:`, `file_store:`, `schemas:`, `audit_log:`, `bus:`, `root:`, `store:` with `ports:`.
2. In the constructor body, read the slice the class needs into instance variables (`@manifest = ports.manifest`, `@bus = ports.event_bus`, etc).
3. Leave `ctx:`, `envelope_io:` / `writer:` / `reader:`, `authorizer:`, `hook_context:` as explicit params (derived collaborators stay explicit).

Audit each class's body for the *narrowest* slice — e.g. `Reads::Get` only needs `ports.manifest` and `ports.file_store`, so the constructor body reads only those two.

- [ ] **Step 5: Hook-RPC bridge for `store:` → `ports:`**

In `lib/textus/hooks/bus.rb#filter_kwargs`, when invoking an RPC callable that declares a `ports:` kwarg, inject the current `ports` value. Update `lib/textus/application/projection.rb#call` to pass both:

- [ ] **Step 1: Write failing tests**

`spec/application/writes/envelope_writer_spec.rb` — `put` writes bytes + appends audit row + returns envelope; `delete` removes file + appends audit row; `move` renames + rewrites name field + appends audit row with `from_key`/`to_key`/`uid` extras; each method raises `EtagMismatch` on mismatched `if_etag`.

- [ ] **Step 2: Create `EnvelopeReader`**

```ruby
class Application::Writes::EnvelopeReader
  def initialize(file_store:, manifest:)
    @file_store = file_store
    @manifest   = manifest
  end

  def read(key)
    res = @manifest.resolver.resolve(key)
    return nil unless @file_store.exists?(res.path)

    raw = @file_store.read(res.path)
    parsed = Entry.for_format(res.entry.format).parse(raw, path: res.path)
    Envelope.build(
      key: key, mentry: res.entry, path: res.path,
      meta: parsed["_meta"], body: parsed["body"],
      etag: Etag.for_bytes(raw), content: parsed["content"],
    )
  end

  def existing_uid(key)
    env = read(key)
    env&.uid
  rescue StandardError
    nil
  end

  def exists?(key)
    @file_store.exists?(@manifest.resolver.resolve(key).path)
  end
end
```

## Risks and rollback

**Risk: deprecation noise overwhelms user hooks.** Mitigation: the
warnings route through `@error_log` (info level), not `Kernel#warn`,
so they appear in the existing hook-error rollup rather than
stderr. Worst case: revert Task 2 step 5 (the bridge) and ship the
Ports rename as a hard break in 0.26.0 instead — the bridge is
opt-in noise, not load-bearing.

**Risk: a spec missed the constructor signature change.** Mitigation:
the full suite (`bundle exec rspec`) is run at the end of each task.
Each task is independently revertible: Task 3 depends on Task 2
which depends on Task 1, but reverting Task 3 alone doesn't break
Tasks 1 or 2.

**Risk: hidden caller of `Manifest#zone_kinds` etc. in user
plugins.** Mitigation: shims forward to the new home for one cycle
*and* emit a warning. Users have until 0.26.0 to update.

**Rollback plan:** Each task is one commit. Reverting individual
tasks in reverse order (3 → 2 → 1) restores the prior state cleanly.
Wire format is unchanged so no store-on-disk migration is needed
either direction.
